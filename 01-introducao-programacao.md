## O que é um Computador?

Um computador é uma máquina que segue instruções. Ele não pensa por conta própria — ele faz **exatamente** o que você manda, nem mais, nem menos. Se você der uma instrução errada, ele vai executar a instrução errada sem questionar.

Pense no computador como um assistente muito obediente, mas que não tem iniciativa própria. Ele precisa que você diga **cada passo** do que deve ser feito, na **ordem correta**.

---

## O que é um Programa de Computador?

Um **programa** é um conjunto de instruções escritas em uma ordem específica que dizem ao computador o que fazer. É como uma **receita de cozinha**:

- A receita tem uma lista de ingredientes (os dados)
- A receita tem passos ordenados (as instruções)
- Se você seguir os passos na ordem certa, o resultado é o prato desejado (o resultado do programa)

Quando você usa um aplicativo no celular, um navegador de internet ou uma calculadora no computador, você está usando um programa que alguém escreveu.

### Exemplos de programas no dia a dia

| O que você usa | O que o programa faz |
|----------------|---------------------|
| Calculadora | Recebe números e operações, calcula o resultado |
| Navegador de internet | Recebe um endereço (URL), busca a página e mostra na tela |
| Aplicativo de mensagens | Recebe texto digitado, envia para outra pessoa, mostra respostas |
| Planilha eletrônica | Organiza dados em linhas e colunas, faz cálculos |

Todos esses programas foram escritos por pessoas — programadores — usando linguagens de programação.

## O que é um Algoritmo?

Um [algoritmo](00-glossario-a-d.md#algoritmo) é uma **sequência de passos ordenados** para resolver um problema ou realizar uma tarefa. Você já usa algoritmos no dia a dia sem perceber!

### Exemplo 1: Receita de café

1. Pegue uma xícara
2. Coloque água no filtro
3. Adicione o pó de café
4. Espere a água passar pelo filtro
5. Sirva o café na xícara

Isso é um algoritmo! Uma sequência de passos, na ordem certa, para chegar a um resultado.

### Exemplo 2: Ir ao mercado

1. Faça uma lista do que precisa comprar
2. Vá até o mercado
3. Para cada item da lista, procure na prateleira e coloque no carrinho
4. Vá ao caixa e pague
5. Volte para casa com as compras

Perceba que a **ordem importa**: você não pode pagar antes de colocar os itens no carrinho, e não pode ir ao mercado sem saber o que precisa comprar.

### Algoritmos na programação

Na programação, um algoritmo é a mesma coisa: uma sequência de instruções que o computador segue para resolver um problema. A diferença é que, em vez de escrever em português, escrevemos em uma **linguagem de programação** que o computador entende.

## O que é Programação?

**Programar** é o ato de escrever instruções (algoritmos) em uma [linguagem de programação](00-glossario-j-p.md#linguagem-de-programação) para que o computador execute uma tarefa.

É como ser um **autor de receitas**: você pensa no que quer que o computador faça, planeja os passos necessários e escreve esses passos em uma linguagem que o computador entende.

### O que é uma Linguagem de Programação?

Nós, humanos, nos comunicamos usando idiomas como português, inglês ou espanhol. Computadores, por outro lado, só entendem **linguagem de máquina** — sequências de zeros e uns (0 e 1) que são impossíveis de ler para nós.

Uma **linguagem de programação** é um meio-termo: uma linguagem que nós conseguimos ler e escrever, e que pode ser traduzida para linguagem de máquina para o computador executar.

Existem centenas de linguagens de programação. Algumas das mais conhecidas:

| Linguagem | Usada para |
|-----------|-----------|
| **Python** | Ciência de dados, automação, web, inteligência artificial |
| **JavaScript** | Sites e aplicações web |
| **Java** | Aplicações empresariais, Android |
| **C** | Sistemas operacionais, jogos, dispositivos |
| **C#** | Jogos (Unity), aplicações Windows |

Neste curso, vamos usar **Python** — uma linguagem conhecida por ser simples, legível e amigável para iniciantes.

## Qualquer Pessoa Pode Aprender a Programar

Se você está lendo isso e pensando "será que eu consigo?", a resposta é: **sim, você consegue**.

Programar não exige ser um gênio da matemática ou ter nascido com algum talento especial. Exige apenas:

- **Curiosidade** — vontade de entender como as coisas funcionam
- **Paciência** — aceitar que errar faz parte do processo (programadores profissionais erram o tempo todo!)
- **Prática** — quanto mais você pratica, mais natural se torna

Não importa sua idade, sua formação ou sua experiência anterior. Pessoas de todas as idades e origens aprendem a programar todos os dias. O mais importante é **não desistir** quando algo parecer difícil — porque vai parecer difícil às vezes, e isso é completamente normal.

## Perguntas Frequentes (FAQ)

**P: Preciso saber matemática para programar?**
R: Não! Para este curso, você só precisa saber as quatro operações básicas (soma, subtração, multiplicação e divisão). Sempre que um conceito matemático aparecer, ele será explicado com calma e com exemplos do dia a dia.

**P: Preciso saber inglês?**
R: Não! Todo o material está em português. Os códigos usam nomes em inglês (é a convenção profissional), mas cada termo será traduzido e explicado nos comentários.

**P: Nunca mexi em um computador direito. Consigo acompanhar?**
R: Sim! Os módulos 03 e 04 explicam passo a passo como usar o terminal e o editor de código. Cada interação com o computador será detalhada.

**P: Quanto tempo leva para aprender a programar?**
R: Depende do seu ritmo, e cada pessoa tem o seu. O importante é estudar com consistência — um pouco por dia é melhor do que muito de uma vez. Não existe pressa.

**P: E se eu não entender algo de primeira?**
R: Isso é completamente normal! Até programadores experientes precisam reler conceitos. Releia o módulo, consulte o [glossário](00-glossario.md) e, se precisar, peça ajuda ao professor. Não existe pergunta boba.

**P: Programar é difícil?**
R: Programar tem seus desafios, como qualquer habilidade nova. Mas com prática e paciência, se torna cada vez mais natural. Pense em quando você aprendeu a cozinhar ou a andar de bicicleta — no início parecia impossível, mas hoje é automático.

**P: Sou muito velho/nova para aprender a programar?**
R: Não existe idade certa para aprender! Pessoas de 15 a 70 anos aprendem a programar todos os dias. O que importa é a vontade de aprender.

**P: Preciso de um computador potente?**
R: Não! Qualquer computador que rode Linux e consiga abrir um editor de texto é suficiente para este curso. Python é uma linguagem leve.

**P: Vou conseguir emprego depois deste curso?**
R: Este curso dá uma base sólida em lógica de programação e Python. Para o mercado de trabalho, você precisará continuar estudando e praticando, mas este é um excelente primeiro passo.

**P: O que é um "bug"?**
R: Bug é o nome que damos a um erro no programa. O nome vem do inglês e significa "inseto" — conta a lenda que um dos primeiros erros de computador foi causado por um inseto real preso dentro da máquina!

**P: Posso estudar no celular?**
R: Você pode ler o material no celular, mas para praticar programação você precisa de um computador com Linux, Python e VSCode instalados.

**P: E se eu errar muito?**
R: Errar é a melhor forma de aprender! Programadores profissionais erram dezenas de vezes por dia. Cada erro é uma oportunidade de entender melhor como as coisas funcionam.

**P: Preciso decorar todos os comandos?**
R: Não! Programadores profissionais consultam documentação e referências o tempo todo. O importante é entender os conceitos — os comandos você pode consultar no [glossário](00-glossario.md).

**P: O que acontece se eu pular um módulo?**
R: Cada módulo depende dos anteriores. Pular um módulo é como pular um degrau da escada — você pode tropeçar nos seguintes. Siga a ordem recomendada.

**P: Posso fazer os exercícios em grupo?**
R: Sim! Estudar em grupo pode ser muito produtivo. Mas tente resolver cada exercício sozinho primeiro, e depois compare com os colegas.

**P: Quanto tempo devo dedicar por dia?**
R: O ideal é pelo menos 30 minutos a 1 hora por dia. Mas qualquer tempo que você dedicar já é um progresso. Consistência é mais importante que quantidade.

**P: O que é "lógica de programação"?**
R: É a habilidade de pensar de forma estruturada para resolver problemas — dividir um problema grande em passos menores e organizá-los na ordem certa. É como planejar uma viagem: você precisa decidir o destino, o meio de transporte, o que levar, etc.

**P: Python é a melhor linguagem para começar?**
R: Python é uma das melhores linguagens para iniciantes por sua sintaxe simples e legível. Mas não existe "a melhor" — o importante é começar com alguma e aprender os conceitos fundamentais.

**P: Vou aprender a criar aplicativos de celular?**
R: Este curso foca em lógica de programação e desenvolvimento com Python (incluindo APIs web). Para criar aplicativos de celular, você precisaria aprender outras tecnologias depois, mas os conceitos que aprende aqui são a base para qualquer área.

**P: E se eu quiser desistir?**
R: É normal sentir vontade de desistir quando algo parece difícil. Quando isso acontecer, faça uma pausa, respire, e volte depois. Releia o módulo com calma. Peça ajuda. Lembre-se: todo programador já se sentiu assim no início.
